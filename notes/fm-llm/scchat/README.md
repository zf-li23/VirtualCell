# scChat 学习笔记

> scChat 是一个**由大语言模型驱动的单细胞 RNA-seq 分析副驾驶**，它不仅执行标准的细胞注释，还能理解**研究上下文**（实验目标、条件描述等），提供假设验证、实验建议等高级功能。采用**多智能体架构** + **RAG 增强的知识图谱**，是单细胞分析从"工具"迈向"合作者"的代表性工作。

### 核心思想

> **单细胞分析不应只是"跑流程"，而应理解"为什么要这样做"。** scChat 能记住你的实验目标和条件，在整个分析过程中提供上下文感知的建议，而不是机械地执行标准 pipeline。

## 2. 模型架构

### 2.1 多智能体编排

scChat 包含 5 个专门 agent：

```
用户查询 + 数据 + 上下文
        │
  ┌─────┴──────┐
  │   Planner   │ ← 解析查询，生成执行计划
  └─────┬──────┘
        │
  ┌─────┴──────┐
  │  Executor   │ ← 执行具体函数（注释/富集/可视化）
  └─────┬──────┘
        │
  ┌─────┴──────┐
  │  Evaluator  │ ← 验证结果，处理错误
  └─────┬──────┘
        │
  ┌─────┴──────┐
  │   Critic    │ ← 检查是否遗漏功能
  └─────┬──────┘
        │
  ┌─────┴──────────┐
  │Response Gen.   │ ← 生成最终回复
  └────────────────┘
```

### 2.2 RAG 增强的知识图谱

使用 Neo4j 图数据库存储生物学知识（CellMarker、GO、KEGG、Reactome），通过检索增强生成（RAG）为 LLM 提供准确的领域知识。

## 3. 核心创新

### 3.1 研究上下文感知

传统工具只回答"这是什么细胞类型？"，scChat 能理解"我在研究免疫疗法耐药性，请帮我分析 T 细胞耗竭标志物"这样的高层次问题。

### 3.2 多智能体编排

完整的 Planner→Executor→Evaluator→Critic→Response 工作流，每个 agent 有明确的职责，支持错误恢复和结果验证。

### 3.3 假设生成和实验建议

超越数据分析，能根据结果提出下一步实验建议——这是向"AI 科学家"迈进的尝试。
