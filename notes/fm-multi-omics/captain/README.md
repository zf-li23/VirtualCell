# CAPTAIN 学习笔记

> CAPTAIN 是一个在超过 400 万具有并发 RNA 和蛋白质测量的单细胞上预训练的多模态基础模型。它首次将大规模预训练范式引入单细胞 RNA + 蛋白联合分析，支持零样本的蛋白质插补和跨物种对齐。

## 1. 模型概述

| 属性 | 描述 |
|------|------|
| **论文** | [CAPTAIN: A multimodal foundation model pretrained on co-assayed single-cell RNA and protein](https://www.biorxiv.org/content/10.1101/2025.07.07.663366v1) |
| **发布日期** | 2025-07 |
| **出版** | bioRxiv |
| **架构** | 双编码器 Transformer（RNA + 蛋白质）+ 跨模态注意力融合 |
| **预训练任务** | 掩码多模态重建（MMR） |
| **输入** | scRNA-seq 表达量 + 表面蛋白质（ADT）表达量 |
| **输出** | 统一多模态细胞嵌入、蛋白质表达预测 |
| **词表** | 基因 + 蛋白质联合词汇表 |
| **参数规模** | ~100M |
| **预训练数据** | scT&P-4M（400 万细胞，CITE-seq） |
| **代码** | [GitHub: iamjiboya/CAPTAIN](https://github.com/iamjiboya/CAPTAIN) |
| **许可** | 待确认 |

### 核心思想

> 用**大规模 RNA+蛋白配对数据**训练一个双编码器 Transformer，让模型同时理解两种模态的"语言"。RNA 编码器初始化自 scGPT 权重，蛋白质编码器借鉴 BLIP 的跨模态思路。通过对两种模态的掩码重建，模型学会了跨模态对应关系。

## 2. 模型架构

### 2.1 整体架构

```
输入: RNA 表达 + 蛋白质(ADT) 表达
          │
          ▼
  ┌───────────────┐    ┌──────────────────┐
  │ RNA 编码器     │    │ 蛋白质编码器      │
  │ (scGPT 初始化) │    │ (BLIP 风格)      │
  │ Transformer   │    │ Transformer      │
  └───────┬───────┘    └────────┬─────────┘
          │                     │
          ▼                     ▼
  ┌──────────────────────────────────────┐
  │     跨模态注意力融合 (Cross-Attention) │
  └──────────────────────────────────────┘
          │
          ▼
  ┌──────────────────────────────────────┐
  │     双重预测头                        │
  │  预测掩码 RNA + 预测掩码蛋白质表达      │
  └──────────────────────────────────────┘
```

### 2.2 核心组件

**RNA 编码器**：基于 Transformer Encoder，权重初始化自 scGPT。

**蛋白质编码器**：独立于 RNA 编码器的 Transformer，设计灵感来自 BLIP 的跨模态预训练。

**跨模态融合**：使用交叉注意力（Cross-Attention）让 RNA 和蛋白质的表示相互促进。

### 2.3 计算优化

- 使用 **FlashAttention-2** 加速大规模序列处理
- 支持在单张 A100 GPU 上完成预训练

## 3. 核心创新

### 3.1 首个大规模 RNA+蛋白联合预训练

CAPTAIN 是已知工作中第一个在**百万级配对数据**上进行 RNA+蛋白联合预训练的基础模型。之前的 scGPT 主要在 RNA 数据上预训练后微调到蛋白任务；CAPTAIN 从零开始在配对数据上预训练。

### 3.2 零样本蛋白质插补

只给 RNA 数据就能预测对应的蛋白质表达。这对只有 RNA 测序但缺少蛋白质数据的实验场景极有价值。

### 3.3 跨物种对齐

在人类和小鼠的配对数据上联合训练，学习跨物种保守的 RNA-蛋白对应关系。

## 4. 数据预处理

| 属性 | 值 |
|------|-----|
| **数据集** | scT&P-4M |
| **技术** | CITE-seq |
| **规模** | 4,000,000+ 细胞 |
| **物种** | 人类 + 小鼠 |

标准 scRNA-seq 预处理流程：质量过滤 → 对数归一化 → HVG 选择。蛋白质表达同样经过标准化处理。

---

## 5. Tokenization 与输入编码

基因通过 Embedding 层映射为向量（与 scGPT 类似）。每种表面蛋白（ADT）通过独立的 Embedding 层映射。表达值通过线性投影与 token 嵌入相加。

---

## 6. 预训练

**预训练目标**：掩码多模态重建（MMR），同时预测掩码的 RNA 表达和蛋白质表达：

$$\mathcal{L} = \mathcal{L}_{\text{RNA}} + \lambda \cdot \mathcal{L}_{\text{Protein}}$$

| 参数 | 值 |
|------|-----|
| 硬件 | NVIDIA A100 GPU |
| 加速 | FlashAttention-2 |
| 优化器 | AdamW |

## 9. 关键概念 Q&A

### Q1: CAPTAIN 和 scGPT 有何不同？

CAPTAIN 专为 RNA+蛋白配对数据设计，RNA 编码器初始化自 scGPT。它的核心优势是**零样本蛋白质预测**，而 scGPT 需要通过微调来适配蛋白任务。

### Q2: CAPTAIN 的局限性？

预训练数据依赖 CITE-seq 技术，覆盖的蛋白种类有限（通常 100-200 种）。对于新型蛋白 panel，零样本性能可能下降。
