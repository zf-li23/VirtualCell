## 2. 模型架构


### 2.1 核心思想

Tabula 挑战了"单细胞数据必须用 Transformer 建模"的主流假设。它将每个细胞视为表格中的一行（基因是列），使用**表格深度学习**架构处理：

```text
细胞表达向量 [g_1, g_2, ..., g_N]
         │
    ┌────┴────┐
    │ 特征编码  │ ← 每个基因作为一列特征
    └────┬────┘
         │
    ┌────┴────────┐
    │ FT-Transformer │ ← 表格数据专用 Transformer
    └────┬────────┘
         │
    细胞嵌入
```

### 2.2 表格 vs 序列

| 维度 | 表格方法 (Tabula) | 序列方法 (Geneformer/scGPT) |
|------|-----------------|--------------------------|
| 数据视图 | 基因作为特征列 | 基因作为序列 token |
| 基因排序 | 不重要 | 至关重要（排序表达） |
| 架构 | FT-Transformer | BERT/GPT |
| 零表达处理 | 天然支持 | 需特殊处理 |



## 3. 核心创新


### 3.1 全新范式：表格而非序列

这是首个系统性地探索"单细胞数据作为表格数据"的工作，挑战了当前主流的序列建模范式。

### 3.2 与序列方法的性能对比

实验证明，表格方法在以下任务上不输甚至优于序列方法：
- 细胞类型注释
- 批次校正
- 基因表达预测

### 3.3 效率优势

表格方法的训练效率通常更高，因为无需处理序列长度带来的 O(N²) 复杂度。



## 4. 数据预处理


标准数据预处理（归一化、对数变换、特征选择），保留零表达值（表格方法天然支持零值）。



## 5. Tokenization 与输入编码


不同于序列方法，Tabula 不需要将基因排序。每个基因是表格的一列，表达值直接作为特征值输入。



## 6. 预训练


自监督掩码策略：随机掩码部分基因的表达值，模型预测被掩码的值。



## 7. 下游任务


| 任务 | 方法 | 性能 |
|------|------|------|
| 细胞类型注释 | 嵌入 + 分类器 | 与序列方法相当 |
| 批次校正 | 嵌入对齐 | 竞争力 |
| 基因表达预测 | 掩码预测 | 良好 |

## 9. 关键概念 Q&A


**Q: 表格方法相比序列方法的核心优势？**
A: ① 无需决定基因排序策略；② 天然处理零表达值；③ 训练效率更高；④ 可解释性更好（特征重要性分析）。

**Q: 为什么之前没人做这个？**
A: 因为 NLP 的巨大成功让人们默认序列模型是处理离散 token 的最佳方式。Tabula 提醒我们：单细胞数据在本质上是表格数据，直接用表格方法可能更自然。
